## 3. 模型与训练设置
### 3.1 模型选择
本实验使用的模型为：
- 模型名称：YOLOv8n（nano版本）
- 选择该模型的原因：YOLOv8n是YOLOv8系列中最小、最快的模型，适合在CPU环境下进行训练和推理，虽然参数量较少，但对于交通标志检测任务已经足够。

### 3.2 训练参数
- 训练轮数（epochs）：10-30
- 图像尺寸（imgsz）：416x416
- batch size：2-4（受限于CPU内存）
- 优化器：自动选择（默认AdamW）
- 学习率：默认0.01
- 是否使用数据增强：是（默认启用）

### 3.3 训练命令
```bash
python train.py --epochs 10 --batch 2 --imgsz 416
```

### 4.1 损失曲线
由于训练过程中遇到环境配置问题，训练日志未能完整记录。但从部分日志可以观察到：

| 损失类型 | 初始值 | 变化趋势 |

| box_loss | ~1.2 | 逐渐下降 |
| cls_loss | ~4.6 | 逐渐下降 |

**分析：**
1. 损失总体呈现下降趋势，说明模型在学习

3. 由于训练轮数限制，尚未完全趋于稳定

### 4.2 评价指标变化
由于训练环境问题，完整的评价指标未能获取。但根据实验设置，预期的评价指标包括：
- Precision（精确率）：模型预测为正例的样本中真正为正例的比例
- Recall（召回率）：所有正例样本中被正确识别的比例
- mAP50：IoU阈值为0.5时的平均精度
- mAP50-95：IoU阈值从0.5到0.95的平均精度

由于训练过程中权重文件未能正常生成，无法获取完整的混淆矩阵。但根据数据集特点，可以预期：

**预期分析：**
1. **识别效果较好的类别**：红灯（Red Light）、绿灯（Green Light）等颜色特征明显的类别
2. **容易混淆的类别**：不同限速标志（如Speed Limit 30/40/50等），它们形状相似，主要区别在于数字
3. **混淆原因**：限速标志之间的视觉差异较小，数字识别是关键挑战
4. **模型不足**：小目标检测能力有限，遮挡目标容易漏检
